## 1. Resumen Ejecutivo

### Estado Actual

OpositaSmart tiene implementados los siguientes flujos de estudio:

| Componente | Estado | Archivo |
|------------|--------|---------|
| Onboarding (5 pasos) | Funcional | `OpositaApp.jsx` |
| Test rapido (5 preguntas) | Funcional | `OpositaApp.jsx` (first-test) |
| HybridSession (FSRS) | Funcional | `HybridSession.jsx` |
| Seleccion de Tema | Parcial | `TemasListView.jsx` |
| Simulacros cronometrados | No implementado | - |
| Flashcards | No implementado | - |
| Repaso de errores | No implementado | - |
| Modo solo lectura | No implementado | - |

### Problemas Principales

1. **Confusion de flujos:** Existen DOS sistemas de quiz paralelos (`first-test` en OpositaApp y `HybridSession`) con comportamientos diferentes
2. **Sin navegacion a temas:** `TemasListView` muestra temas pero `onTopicSelect` solo hace console.log
3. **Sin modos de estudio claros:** Usuario no puede elegir entre practica rapida, simulacro o repaso
4. **FSRS no testeable:** Developer no puede verificar que el algoritmo funciona correctamente

### 4.4 Flashcards y FSRS

**SI, flashcards alimentan el mismo sistema FSRS.**

Adaptacion del algoritmo para flashcards:

```javascript
// Flashcard tiene 2 botones en lugar de 4 opciones
const flashcardRatings = {
  'show_answer': null,    // Solo muestra, no califica
  'again': 1,             // "No lo sabia" â†’ intervalo = 1 dia
  'good': 3,              // "Lo sabia" â†’ intervalo * ease
  'easy': 4               // "Muy facil" â†’ intervalo * ease * 1.3
};

// Para flashcards simplificadas (2 botones):
// "Repasar" â†’ again (1)
// "Lo se" â†’ good (3)
```

### 5.3 Implementacion Tecnica

```javascript
// src/components/dev/FSRSDebugPanel.jsx

import { useState, useEffect } from 'react';
import { supabase } from '../../lib/supabase';
import { useAuth } from '../../contexts/AuthContext';
import { getDueReviews, getUserProgress } from '../../services/spacedRepetitionService';
import { calculateNextReview, QuestionState } from '../../lib/fsrs';

export default function FSRSDebugPanel() {
  const { user } = useAuth();
  const [simulatedDate, setSimulatedDate] = useState(new Date());
  const [dueQuestions, setDueQuestions] = useState([]);
  const [allProgress, setAllProgress] = useState([]);
  const [selectedQuestion, setSelectedQuestion] = useState(null);

  // Simular fecha
  const simulateDate = (daysOffset) => {
    const newDate = new Date();
    newDate.setDate(newDate.getDate() + daysOffset);
    setSimulatedDate(newDate);
  };

  // Calcular "due" con fecha simulada
  const calculateDueWithSimulation = (progress) => {
    if (!progress.next_review) return true;
    return new Date(progress.next_review) <= simulatedDate;
  };

  // Cargar datos
  useEffect(() => {
    async function loadData() {
      if (!user?.id) return;
      const progress = await getUserProgress(user.id);
      setAllProgress(progress);

      // Filtrar due con fecha simulada
      const due = progress.filter(p => calculateDueWithSimulation(p));
      setDueQuestions(due);
    }
    loadData();
  }, [user?.id, simulatedDate]);

  // Forzar pregunta como due
  const forceDue = async (questionId) => {
    await supabase
      .from('user_question_progress')
      .update({ next_review: new Date().toISOString() })
      .eq('user_id', user.id)
      .eq('question_id', questionId);
  };

  // Reset progreso
  const resetProgress = async (scope, scopeId = null) => {
    let query = supabase
      .from('user_question_progress')
      .delete()
      .eq('user_id', user.id);

    if (scope === 'tema' && scopeId) {
      // Necesita join con questions para filtrar por tema
      const { data: questionIds } = await supabase
        .from('questions')
        .select('id')
        .eq('tema', scopeId);
      query = query.in('question_id', questionIds.map(q => q.id));
    } else if (scope === 'question' && scopeId) {
      query = query.eq('question_id', scopeId);
    }

    await query;
  };

  return (
    // ... UI del panel
  );
}
```
